TBits 6.0 Help   

New Features in TBits V6.0

There are many new and modified features in Version 6.0 of TBits. A synopsis of some of the highlights follows.

Adding and Updating Requests

  • Inline Update Commands: The former Email Update Commands (EUCs) can now be used within the web interface, and have been renamed Inline Update Commands (IUCs). There are many improvements, including the new /link descriptor. There is now support for minimal matching of descriptors.
  • Transfer Request Improvements: A number of changes made to improve the workflow when a request is transferred.
  • Office Field: This field can be used to specify the location from which a request was logged.
  • Disable Email: You can now disable email for yourself for an entire business area.
  • Improvements in Saving Drafts: More fields of drafts are now saved, including many of the list type fields. Special characters in drafts are also now saved.
  • Private: "Confidential" has been renamed back to "private".

Searching

  • Read/Unread: You can now search based on the read/unread setting of a request.
  • Date Selection: You can search based on the date a request was submitted or last updated, or based on the due date.
  • Office Field: You can search for the office where the request was logged.
  • Copy Search URL: You can copy your search URL to the Windows clipboard with a single click.
  • DQL and the Query Builder: The Advanced Search Query Builder and DQL now include support for these features:
    • Specify that a field has no value: is empty or -has:<descriptor>
    • Specify that a field has a value: is non empty or has:<descriptor>
    • Search for members of mailing lists: including members of or <user_name_field>: memberof : <mailing_list>
    • Search to exclude members of mailing lists: excluding members of or -<user_name_field>: memberof : <mailing_list>
    • DQL has added descriptors disphdr, sortorder, sortfield to control the display and sorting of fields
    • Descriptors for fields and for list type field values can be viewed for the business area.
  • Search Label Names: A number of the functions available when searching have more easily understandable names.
  • Save Fields when Switching BA: When you switch business areas when using Search, most text fields and common list type fields preserve their value.

Viewing Requests

  • Display of Parent/Subrequest Trees: The display of parent/subrequest relationships has been significantly improved.
  • My Results Page Performance: The response time of the My Requests page has been improved significantly.
  • Summary Differences: It is now possible to view the differences between the current summary and the previous summary of a request. This is an experimental feature; expect improvements in future releases.
  • Saving File Attachments: A save link is now provided for file attachments.
  • Smart Tag Improvements: Smart tags now get a tooltip with the subject of the request. Support has also been added for the notation req# and request#.
  • Update Log: The update log has been improved to allow easier understandability of any changes made. For example:
    [ Subscribers: reiss, konetil, krovid, sethig ]
  • History Bar Improvements: The Request History bar has been streamlined to reduce unnecessary information.
  • Firefox and Attachments: Support has been added to allow the opening of PNG images and HTML attachments in Firefox.

Admin

  • Mailing Lists: Assign authorization access and roles based on mailing lists.
  • Business Area Information: You can view details for the business area, such as the administrators and a list of supported IUC descriptors.
