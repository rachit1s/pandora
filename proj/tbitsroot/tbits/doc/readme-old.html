Introduction

Thank you for choosing tBits, version 6.0.

tBits is the next generation web-based issue tracking system that combines the power of workflow, database systems, advanced searching technology and the convenience of email systems.

tBits, initially designed for bug tracking, has evolved to become a very generic tool - essentially a simple database with a web front having the capability to track the history of each row and field that can be defined by the user. To effectively search the database, tBits comes with its own search query language that can search even inside attachments like pdf, doc, xls, ppt.

The operations on the system can be performed either by using a browser or an email client using defined grammar. The intelligence in the system is given by a powerful inbuilt Workflow Rule Engine and notification rules mechanism.

Features

  • Organizational Workflow Rules / Procedures are mapped into tBits for issues/requests.
  • Advanced searching of data repository using an advanced Data Query Language
  • Powerful search that enables you to search inside excel, word, text, pdf etc. document formats.
  • Complete email integration enabling you to raise and respond to requests/issues from your email client boxes rather than going to the browser.
  • Completely web-based & Operating System Independent.
  • Intuitive, Attractive and light-weight AJAX enabled interface.
  • Can be exposed to external clients and use as CRM.
  • Internalize organizational learning from issues/requests by using tBits as a Knowledge Base Management System.
  • Equipped with smart tools including Spell Checker, Summary Differentiator, Smart Tags.
  • A generic customizable, pluggable, programmable object tracking system with the freedom to define your own rules, fields & external API's.

System Requirements

tBits uses Microsoft SQL Database. If you don't have MS SQL server installed, please download the express edition which is free. JDK 1.5 or higher must be installed on your system.

Running the installer

Once you are done with the database creation, start the installer.jar by clicking on it or by using the command java -jar installer.jar. The installer wizard will take you to the following steps:
  • Welcome Screen

    • Click 'Next'.

  • Message

    • Read the details carefully and click 'Next'.

  • License Agreement

    • Select 'I accept' and hit 'Next'.

  • JDK Path

    • It is not path to JRE. Select the location where JDK is installed and hit 'Next'.

  • Installation Path

    • Select the installation path and hit 'Next'. This is the location where tBits gets installed. We will be referring to it as INSTALL_PATH.

  • Database Connection Parameters

    • Enter the name of database server which will be your machine name by default. Leave the database name filled with "tbits" (without quotes). In the User credentials section, input the user name (and its password) which will be used by tBits to connect to database. The admin credentials are not required since the installer doesn't create the database. In future versions, the installer will create the database.

  • Domain Controller

    • This is required if you want to use tBits to authenticate using Domain Controller. Since the default installation uses the tBits - Internal authentication (A form based authentication), you can just ignore this form and click 'Next'.

    In case you plan to integrate with Domain Controller, follow these steps:

    1. Specify the name of the server which acts as the domain controller in your network and the LDAP server which will be used for user synchronization.
    2. Change the authentication filter. Edit web.xml located in INSTALL_PATH/build/webapps/WEB-INF. Uncomment the filter block that uses jcifs.http.NtlmHttpFilter and comment the filter block that uses the class transbit.tbits.authentication.AuthenticationFilter.
    3. Switch-on the LDAP account synchronization. Edit INSTALL_PATH/build/etc/tbits-quartz-jobs.xml and uncomment the job block with "AD2TBitsSync" as the job name. If you don't select this, you can still use the system but you will have to add accounts using the 'All Users' Tab in the Administration Panel of tBits.

  • SMTP Server

    • Since tBits would be sending mails to various emails ids, you need to configure an outgoing mail server. Type the name of SMTP server. If your server requires authentication check "Requires Authentication" and provide the login name and password.

    NOTE: If your SMTP Server does not accept connections at 25 port, the installer will not proceed. In such a case, just specify any mail server which accepts connections at 25 port and change the settings by editing INSTALL_PATH/build/tbits.properties

  • tBits General Configuration

    • This is the general configuration which can be skipped. You might want to change the Notification Recipient Mail Id. Click 'Next'.

  • Installation of tBits - Packs

    • Just click 'Next'. This will install the tBits. It will take around five minutes. This will be followed by the shortcut creation, just click 'Next' and you will be done.

Configuration

In case you have more requirements, for example, you want the attachments to be stored somewhere else, you would need to edit the configuration. The configuration files located in the INSTALL_PATH/build/etc folder are self descriptive. The templates are located in INSTALL_PATH/build/web. You might want to put the logo of your company at the main screen.

Launching tBits

The Installer must have created the shortcuts on the desktop. Double click on "Start tBits Server" icon and it will start tomcat hosting tBits as well as the scheduler which executes the jobs. Once tomcat starts, double click on the "tBits" shortcut located on the desktop. It will prompt you to enter the login name and password. Just enter 'root' (without quotes) as login and 'root' (without quotes) as password. For more details on using tBits, click on 'tBits Help' on the login screen or on 'Help' located at the top right on the tBits main screen.
